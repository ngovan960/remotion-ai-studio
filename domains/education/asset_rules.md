# Education Asset Rules

## Image Assets

### Required Types
- **Slides**: Presentation-style visuals
- **Diagrams**: Conceptual illustrations
- **Code Screenshots**: Syntax-highlighted code
- **Educational Illustrations**: Teaching aids
- **Stock Photos**: Relevant, professional images
- **Icons**: Universal symbols and markers

### Image Quality Standards
- Minimum resolution: 1920x1080
- Color profile: sRGB
- Format: PNG for graphics, JPEG for photos
- No watermarks or logos
- Consistent style throughout

### Image Sources
- Unsplash (free license)
- Pexels (free license)
- Educational databases
- Custom illustrations
- Generated diagrams

## Video Assets

### Screen Recordings
- Resolution: 1920x1080 minimum
- Frame rate: 30 fps
- Cursor: Visible, highlighted
- Actions: Deliberate, clear
- Annotations: When helpful

### Generated Video
- Code animations with syntax highlighting
- Diagram animations showing concepts
- Text animations for emphasis
- Transition animations between topics
- Progress animations for exercises

## Slide Assets

### Slide Elements
- **Title**: Clear, descriptive
- **Bullet Points**: Concise, readable
- **Images**: Relevant, high-quality
- **Code**: Syntax-highlighted
- **Diagrams**: Labeled, annotated

### Slide Design
- **Layout**: Consistent, organized
- **Colors**: Professional, accessible
- **Fonts**: Readable, appropriate
- **Spacing**: Adequate, not crowded
- **Branding**: Consistent throughout

## Diagram Assets

### Diagram Types
- **Flowcharts**: Process visualization
- **Mind Maps**: Concept relationships
- **Hierarchies**: Structure organization
- **Timelines**: Sequence of events
- **Comparisons**: Side-by-side analysis

### Diagram Elements
- **Shapes**: Clear, meaningful
- **Lines**: Connecting relationships
- **Labels**: Descriptive text
- **Colors**: Categorical coding
- **Icons**: Visual reinforcement

## Code Assets

### Code Display
- **Font**: Monospace (Consolas, Monaco)
- **Size**: Readable at all scales
- **Highlighting**: Syntax-appropriate
- **Line Numbers**: When relevant
- **Annotations**: Explanatory comments

### Code Types
- **Snippets**: Short examples
- **Full Programs**: Complete examples
- **Diffs**: Before/after comparisons
- **Output**: Execution results
- **Errors**: Common mistakes

## Audio Assets

### Voiceover
- **Quality**: Professional, clear
- **Pace**: Moderate, adjustable
- **Tone**: Enthusiastic, encouraging
- **Clarity**: Every word intelligible
- **Consistency**: Same throughout

### Music Selection
- **Genre**: Ambient, lofi, minimal
- **Mood**: Focused, calm, positive
- **Tempo**: 70-90 BPM
- **Volume**: Background level
- **Loop**: Seamless repetition

### Sound Effects
- **Transitions**: Subtle whooshes
- **Success**: Positive chimes
- **Errors**: Gentle alerts
- **Notifications**: Informational sounds
- **Ambience**: Study atmosphere

## Text Assets

### Typography
- **Font**: Clean, readable sans-serif
- **Size**: Large enough for all devices
- **Color**: High contrast
- **Style**: Consistent throughout
- **Animation**: Subtle, purposeful

### Text Content Rules
- **Clarity**: Simple, direct language
- **Conciseness**: Avoid unnecessary words
- **Accuracy**: Fact-checked content
- **Accessibility**: Readable by all
- **Consistency**: Same terminology

## Asset Organization

### Directory Structure
```
assets/
├── slides/
│   ├── templates/
│   ├── backgrounds/
│   └── elements/
├── diagrams/
│   ├── flowcharts/
│   ├── mind_maps/
│   └── hierarchies/
├── code/
│   ├── snippets/
│   ├── examples/
│   └── output/
├── images/
│   ├── stock/
│   ├── illustrations/
│   └── icons/
├── audio/
│   ├── voiceover/
│   ├── music/
│   └── sfx/
└── text/
    ├── titles/
    ├── labels/
    └── annotations/
```

### Naming Convention
```
[domain]_[type]_[topic]_[variant].[ext]
Example: education_slide_python_functions_01.png
```
