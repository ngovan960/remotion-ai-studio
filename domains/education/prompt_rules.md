# Education Domain Prompt Rules

## Core Principles
- Teach clearly and effectively
- Use visual aids to reinforce learning
- Break complex topics into digestible parts
- Encourage engagement and retention

## Prompt Structure

### Scene Description Format
```
[SCENE_TYPE]: [TOPIC] + [LEARNING_OBJECTIVE] + [AUDIENCE_LEVEL]
Visual: [specific_visual_elements_for_teaching]
Camera: [movement_and_angle_for_clarity]
Audio: [voiceover_and_supporting_sounds]
Duration: [seconds]
```

### Example Prompts

**Introduction Scene:**
```
Introduction: Python Functions + Learn to create reusable code + Beginners
Visual: Title slide with Python logo, clean design, learning objectives listed
Camera: Static with subtle zoom on key points
Audio: Clear voiceover introduction, minimal background music
Duration: 8s
```

**Code Demonstration:**
```
Code Demo: Function syntax + Show proper function structure + Intermediate
Visual: Code editor with syntax highlighting, line-by-line reveal
Camera: Static, focused on code area
Audio: Voiceover explaining each line, keyboard sounds
Duration: 12s
```

**Concept Explanation:**
```
Concept: Variables and data types + Understand different data types + Beginners
Visual: Animated diagram showing data types as containers
Camera: Slow pan across diagram
Audio: Voiceover with examples, subtle background music
Duration: 10s
```

**Practice Exercise:**
```
Exercise: Write a function + Apply learned concepts + Beginners
Visual: Interactive code editor with hints, progress tracker
Camera: Static with focus on active area
Audio: Encouraging voiceover, success sounds
Duration: 15s
```

## Style Guidelines

### Visual Language
- Use clear, readable fonts
- Maintain consistent color scheme
- Include visual hierarchy
- Use icons and symbols appropriately
- Show code with syntax highlighting

### Teaching Techniques
- Start with overview, then details
- Use examples to illustrate concepts
- Provide practice opportunities
- Summarize key points
- Use repetition for reinforcement

### Pacing
- Allow time for comprehension
- Pause at key points
- Repeat important concepts
- Use visual breaks between sections

## Content Organization

### Lesson Structure
1. **Introduction**: What we'll learn
2. **Overview**: Big picture context
3. **Main Content**: Step-by-step instruction
4. **Examples**: Concrete demonstrations
5. **Practice**: Hands-on application
6. **Summary**: Key takeaways
7. **Next Steps**: What to learn next

### Visual Hierarchy
- **Titles**: Large, bold, clear
- **Subtitles**: Medium, descriptive
- **Body Text**: Readable, concise
- **Code**: Monospace, highlighted
- **Diagrams**: Labeled, annotated

## Engagement Techniques

### Interactive Elements
- **Progress Indicators**: Show completion
- **Checkpoints**: Verify understanding
- **Quizzes**: Test knowledge
- **Challenges**: Apply skills

### Visual Interest
- **Animations**: Subtle, purposeful
- **Transitions**: Smooth, logical
- **Color Coding**: Consistent meaning
- **Icons**: Universal symbols

### Audio Engagement
- **Voiceover**: Clear, enthusiastic
- **Sound Effects**: Reinforce actions
- **Music**: Support focus
- **Pacing**: Match content complexity

## Forbidden Patterns
- Don't overwhelm with information
- Avoid cluttered visuals
- Don't skip foundational concepts
- Avoid jargon without explanation
- Don't rush through complex topics
